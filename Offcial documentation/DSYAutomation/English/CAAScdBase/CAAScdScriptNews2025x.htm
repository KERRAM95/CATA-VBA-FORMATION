What's New with 3DEXPERIENCE R2025x Automation
Technical Article

Abstract

This article presents Automation objects, methods, and properties that are added since 3DEXPERIENCE R2025x GA to enhance V6 Automation capabilities, support functional evolutions, and correct defects. Some of those modifications may already exist in 3DEXPERIENCE R2024x maintenance fixes. A few properties and methods might have been modified or removed. VB and VBScripts code using those methods may have to be modified. VB projects using the corresponding objects must be recompiled.

Consult About Microsoft Automation Languages, Debug, and Compatibility for details on manual adjustments that may be required for existing VBA and VSTA projects to be usable on a new release.

Modified Objects, Methods and Properties
Object SimDesignSpace has been renamed into SimDesignEnvelope.
The following object has been moved into new type libraries. This has no impact on vbscript/CATScript macros and should have no impact on VBA project as well, however, in case of problem, apply the procedure described in the Inter Release Compatibility paragrah:
  • SimScalarField, moved from SMAMpaBase TypeLib (SMAMpaBaseIDLItfTypeLib.tlb) into SMAMpaFoundationIDLItf TypeLib (SMAMpaFoundationIDLItfTypeLib.tlb).
  • SMAIAMpaSpaceTimeField, moved from SMAMpaBase TypeLib (SMAMpaBaseIDLItfTypeLib.tlb) into SMAMpaFoundationIDLItf TypeLib (SMAMpaFoundationIDLItfTypeLib.tlb).
  • SimSymmetricTensorField, moved from SMAMpaBase TypeLib (SMAMpaBaseIDLItfTypeLib.tlb) into SMAMpaFoundationIDLItf TypeLib (SMAMpaFoundationIDLItfTypeLib.tlb).
  • SimConnectorTranslation object has been moved from SMAMpaStructuralModelIDLItf type library into SMAMpaFoundationIDLItf TypeLib (SMAMpaFoundationIDLItfTypeLib.tlb).
