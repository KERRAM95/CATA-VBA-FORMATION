Getting Started with Python
 

This task shows you how to get started with Python. It will show you how to use a scripting language to access CAA Automation objects to capture your own know-how and to increase your productivity. You can customize applications to automate repetitive tasks, and to make it fit your own process.

You will discover here how to create and execute a Python script through an example dealing with creating a mesh part.
Before you begin:
  • You should first log in to the 3DEXPERIENCE platform.
  • You should open a simulation to access the python command.
  • The apps that make up the the 3DEXPERIENCE platform share the same object model [1]which can be accessed, as well as their own objects, by scripts written in Python. Refer to About Automation Languages, Debug, and Compatibility [2] to learn about the languages supported.
  • You can then find information about the scripting languages and environments, in Running and Editing Macros. This is located in the product documentation, Social and Collaborative Apps, Collaborative Apps Common Services, Native Apps Common Services, Other Services.

Creating the Script

  1. Create a .py file and edit it with any editor, and write in it the following script.

  2. Accessing the CATIA application object, and the ENUM object for all enumeration constants.

    	from com3dx import *
	CATIA = get3dxClient()
	ENUM = win32com.client.constants
	...

  3. Retrieving the active simulation and its product.

    1. As a first step, retrieve the active editor.

      	...
	myEditor = CATIA.ActiveEditor
	...

    2. Then retrieve the product service.

      	...
	myProdService = myEditor.GetService("PLMProductService")
	...

    3. Retrieve the PLMEntities collection object.

      	...
	myEntities = myProdService.EditedContent
	...

    4. Now retrieve the PLMEntity.

      	...
	myEntity = myEntities.Item(1)
	...

    5. Get the simulation Root.

      	...
	MySimulationRoot = myEntity
	...

    6. Now retrieve the Model.

      	...
	myModel = MySimulationRoot.Model
	...

  4. Retrieving the Publication.

    1. Get first the list of all publications.

      	...
	myPublications = myModel.Publications
	...

    2. Then retrieve the publication needed to use as support.

      In our sample, we need a publication of a part body to create a support for our mesh

      	...
	myPublication = myPublications.GetItem("Part_Body_Publication_Name")
	...

  5. Creating a Mesh.

    1. Get the product representation factory.

      	...
	myPrdRepFactory = myModel.GetItem("SimPrdRepFactory")
	...

    2. Create the product representation.

      	...
	myRepRef = myPrdRepFactory.CreatePrdRep("FEM")
	...

    3. Get the FEM Root and add the associated representation.

      	...
	myFEMRoot = myRepRef.GetItem("SimFemRoot")
	myFEMRoot.AddAssociatedRep(myPublication)
	...

    4. Get the mesh set and add the mesh part.

      	...
	myMeshSet = myFEMRoot.GetSet("SimNodesElements")
	myMeshParts = myMeshSet.MeshParts
	myMeshPart = myMeshParts.Add("CATFmtOctree3DRulesMesher")
	myMeshPart.AddSupport(myPublication)
	myMeshPart.Update
	...

Running the Script

  1. Run the Python command.

  2. Select your python file.

  3. Uncheck the two options.

  4. Click OK to run the script.

Functions in Python

If you want to split the script in different function for each of the steps 2 to 5 described in Creating the Script, you should define a function for each.

Below is an example for the step 5) "Creating a mesh".

  1. At the end of the step 4, call the function "CreateMesh" with myModel and myPublication as inputs.

    	...
	CreateMesh(myModel,myPublications)

  2. Define the CreateMesh function at the top of the script.

    	def CreateMesh(myModel, myPublications):
		myPrdRepFactory = myModel.GetItem("SimPrdRepFactory")
		myRepRef = myPrdRepFactory.CreatePrdRep("FEM")
		myFEMRoot = myRepRef.GetItem("SimFemRoot")
		myFEMRoot.AddAssociatedRep(myPublication)
		myMeshSet = myFEMRoot.GetSet("SimNodesElements")
		myMeshParts = myMeshSet.MeshParts
		myMeshPart = myMeshParts.Add("CATFmtOctree3DRulesMesher")
		myMeshPart.AddSupport(myPublication)
		myMeshPart.Update
	...

Enumerations in Python

The enumeration values defined for VBA are available in Python. Below is an example of how to use enumerations in Python.

  1. Retrieve the ENUM object.

    	from com3dx import *
	ENUM = win32com.client.constants
	...

  2. Retrieve the enumeration values from the ENUM object.

    	...
	myAxis = myCentrifugalForce.Line
	myAxis.AxisType = ENUM.SimAxisGeometric
	...

Typename in Python

The 3DEXPERIENCE objects available in Python have type information available using the typename function. Python includes a builtin type function, but this is not useful for 3DEXPERIENCE objects. Below is an example of how to use typename in Python.

  1. Retrieve the CATIA Application object.

    	from com3dx import *
	CATIA = get3dxClient()
	...

  2. Retrieve the type information from the CATIA Application object.

    	typename(CATIA)
	type(CATIA)
	...

    This produces the output:

    	'Application'
	<type 'Instance'>

Python Settings

  1. Open Preferences

  2. Go to the Physics Simulation / Python panel.

  3. Choose a Python interpreter.

    • Configured Python Installation is available for customers to run their own custom installation of python. This is recommended for customers that need 3rd party packages.
    • Discontinued Python 2.7 Installation is discontinued. Customers requiring this python release can configure a custom installation.
    • Discontinued Python 3.7 Installation is discontinued. Customers requiring this python release can configure a custom installation.
    • Default Python 3.10 Installation is the default and recommended choice. It requires no extra installation, or configuration.

  4. Optional: Configure your own installed Python interpreter.

    Executable Path is the full filename path of the python interpreter or virtual environment installation. This field cannot be empty.

    Library Path is used to set the PYTHONPATH environment variable. It is set as a series of paths seperated by the ; character. The Python Standard Library is found where the interpreter is installed and does not need to be included here. Most 3rd party packages are installed in the interpeter directory and do not need to be included here. The primary usage of this field is to allow Python to find modules developed by a customer that are imported by their scripts. This field can be empty.

    Editor Path is the command line to launch the 3rd party editor or integrated development environment (IDE) to be used to run or debug Python scripts. If this field is left empty, the default Idle editor from the python installation under Executable Path will be used. The editor will be launched using the python script as the final command line argument. For example, if your Editor Path is c:\MyEditor.exe, and your script name is c:\MyScript.py, the editor will be launched as "c:\MyEditor.exe c:\MyScript.py". Important: the 3rd party editor must be configured to use the same python interpreter as 3DEXPERIENCE. This field can be empty.
