Quick Reference

CATKinMechanismWkb

Role: Setting to manage Mechanical Systems Design options

Detailed role: Mechanical Systems Design setting contains all options necessary to Mechanical Systems Design commands

The CATKinMechanismWkb repository has the following characteristics:

  • Primary Tab Page: /Mechanical Systems/Mechanical Systems Design/General
  • Repository File: CATKinMechanismWkb
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1
  • Root tab page /Mechanical Systems/Mechanical Systems Design/General reused as /Mechanical Systems/Mechanical Systems Design/General

CATKinMechanismWkb setting repository attributes:

  • MechanismManagementForMotion
    Role: Attribute to create and manage mechanisms compatible with Motion Apps.
    Detailed role: Create and manage mechanisms compatible with Motion Apps.
    • Type: boolean
    • Default value: true
    • Default Value Role: By default mechanism management for Motion Apps
  • MechanismUpdation
    Role: Controls weather an engineering connection will be added to existing mechanism or not
    Detailed role: According to this attribute's value, an engineering connection will be added to existing mechanism or not
    • Type: boolean
    • Default value: false
    • Default Value Role: by default newly created enginnering connection will not be added to current mechanism, if any, or to first mechanism, otherwise
  • MechanismUpdationNotif
    Role: Controls weather mechanism update warning needs to be displayed with newly created engineering connection or not
    Detailed role: According to this attribute's value, warning to add a newly created engineering connection will be displayed or not
    • Type: boolean
    • Default value: true
    • Default Value Role: by default message to add enginnering connection to current mechanism, if any, or to first mechanism, otherwise, will be thrown
