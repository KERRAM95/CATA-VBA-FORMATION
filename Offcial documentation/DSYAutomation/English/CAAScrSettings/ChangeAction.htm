ChangeAction Setting Repository
Quick Reference

ChangeAction

Role: Change Action options

Detailed role: This option enables to modify Change Action behaviors.

The ChangeAction repository has the following characteristics:

  • Primary Tab Page: /ENOVIA V5 VPM/Change Action
  • Repository File: ChangeAction
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

ChangeAction setting repository attributes:

  • AutoRefresh
    Role: Is Auto-Refresh activated for CA page
    • Type: boolean
    • Default value: false
    • Default Value Role: By default, Auto-Refresh is deactivated
  • ChangeMgmtCheck
    Role: Change Management Check for warnings
    Detailed role: This option allows the user to set the Preference to allow to show warnings.
    • Type: boolean
    • Default value: false
  • DisplayPreferences
    Role: Display Mode At Start
    Detailed role: This option allows the user to set the Default Display type
    • Type: integer
    • Default value: 1
    • Default Value Role: By default the type is Title
    • Values:
      0 The default type is Name
      1 The default type is Title
  • EnableTileView
    Role: Change Management Check to enable tile view
    Detailed role: This option allows the user to set the Preference to allow to enable Tile view in proposed view and realized view in change action.
    • Type: boolean
    • Default value: false
  • IgnoreChangeControl
    Role: Change Management Check to Ignore Change Required for Authoring Operations on configured structure
    Detailed role: This option allows the user to set the Preference to allow to Ignore Change Required for Authoring Operations on configured structure.
    • Type: boolean
    • Default value: false
  • MultiCA
    Role: Change Management Check for multi-CA mode
    Detailed role: This option allows the user to set the Preference to allow to enable work with associated change action.
    • Type: boolean
    • Default value: false
