RefineImplItemsSettings Setting Repository
Quick Reference

RefineImplItemsSettings

Role: Refinement options

Detailed role: These settings allow to specify which facet can be seen during implemented items refinement

The RefineImplItemsSettings repository has the following characteristics:

  • Primary Tab Page: /ENOVIA V5 VPM/Action
  • Repository File: RefineImplItemsSettings
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

RefineImplItemsSettings setting repository attributes:

  • ListAttributes
    Role: List of the attributs used as facet in refine implemented items
    • Type: string
    • Default value: ProgressStatus/MainOperation
  • ListAttributesCusto
    Role: List of the attributes used as facet in refine implemented items. Define by client
    • Type: string
    • Default value:
  • ListAttributesCustoID
    Role: List of the attributes ID used as facet in refine implemented items. Define by client
    • Type: string
    • Default value:
  • NumberOfAttributes
    Role: stock the number of saved custom attributes
    • Type: integer
    • Default value: 0
    • Default Value Role: By default, 0 attributes are saved
  • ToUpdate
    Role: Allow to know if a modification has been made in Option->tools
    • Type: integer
    • Default value: 0
    • Default Value Role: By default, it's not to update
