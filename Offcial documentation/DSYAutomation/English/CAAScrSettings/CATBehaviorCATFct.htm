CATBehaviorCATFct

Detailed role: Setting to handle Automated Design Process

The CATBehaviorCATFct repository has the following characteristics:

  • Primary Tab Page: /Knowledgeware/Automated Design Process/Automated Design Process
  • Repository File: CATBehaviorCATFct
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1
  • Setting controller:
    • Framework_CPP: CATOBMInterfaces
    • Framework_IDL: CATOBMInterfaces
    • Interface_CPP: CATIBKTBehaviorSettingAtt
    • Interface_IDL: CATIABehaviorSettingAtt
    • LateType: CATBKTBehaviorSettingCtrl

CATBehaviorCATFct setting repository attributes:

  • CHECK_CFGS
    Role: DPC checks available Applicative Configuration at session startup
    Detailed role: Specifies whether DPC checks for available Applicative Configuration at session startup to possibly deploy one.
    • Type: unsigned integer
    • Default value: 0
  • DEFINE_BEHAVIOR
    Role: DPC is working in expert or user mode
    Detailed role: Specifies wether DPC is working in expert or all user mode.
    • Type: unsigned integer
    • Default value: 1
  • HIDE_ATTRIB
    Role: DPC hides neutral attributes
    Detailed role: Specifies whether neutral attributes of technological types must be shown or hidden
    • Type: unsigned integer
    • Default value: 1
  • KACKAX_LICENSING
    Role: Disables the KAC license
    Detailed role: Disables the KAC license.
    • Type: unsigned integer
    • Default value: 0
  • OPERATION_MESSAGES
    Role: DPC is showing operation messages
    Detailed role: Specifies wether Type variables are shown in specification tree
    • Type: unsigned integer
    • Default value: 0
  • SHOW_VARIABLE
    Role: DPC is showing variable
    Detailed role: Specifies wether Type variables are shown in specification tree
    • Type: unsigned integer
    • Default value: 1
