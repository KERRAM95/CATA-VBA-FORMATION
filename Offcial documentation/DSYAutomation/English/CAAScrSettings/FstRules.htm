FstRules Setting Repository
Quick Reference

FstRules

Role: These setting Parameters define the rules to be applied for fastener consistency checks

Detailed role: Enable the user to specify the Rules which are to be activated

The FstRules repository has the following characteristics:

  • Primary Tab Page: /Infrastructure/FastenerInfrastructure/Rules
  • Repository File: FstRules
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

FstRules setting repository attributes:

  • CustoFSTRule_*
    Role: :To apply custom defined rules
    • Type: boolean
    • Default value: false
  • FstFBECompass
    Role: :Enable rule to check that distance between the current fastener (border) and the flange edge is greater than a specified distance
    • Type: boolean
    • Default value: false
  • FstFBECompassCriticality
    Role: Criticality for rule to check that distance between the current fastener (border) and the flange edge is greater than a specified distance
    • Type: string
    • Default value: Medium
    • Values:
      High Higher criticality
      Medium Moderate criticality
      Low Lower criticality
  • FstFCECompass
    Role: :Enable rule to check that distance between the current fastener (origin) and the flange edge is greater than a specified distance
    • Type: boolean
    • Default value: false
  • FstFCECompassCriticality
    Role: Criticality for rule to check that distance between the current fastener (origin) and the flange edge is greater than a specified distance
    • Type: string
    • Default value: Medium
    • Values:
      High Higher criticality
      Medium Moderate criticality
      Low Lower criticality
  • FstFCLSCompass
    Role: :Enable rule to check the fastener geometry in clash with other geometry
    • Type: boolean
    • Default value: false
  • FstFCLSCompassCriticality
    Role: Criticality for rule to check the fastener geometry in clash with other geometry
    • Type: string
    • Default value: High
    • Values:
      High Higher criticality
      Medium Moderate criticality
      Low Lower criticality
  • FstFCLSIgnoreHiddenMode
    Role: Indicates whether objects in hidden mode are to be ignored for FCLS rule computation
    • Type: boolean
    • Default value: true
  • FstFFLCCompass
    Role: :Enable rule to check the fastener length is same as thickness of stack parts
    • Type: boolean
    • Default value: false
  • FstFFLCCompassCriticality
    Role: Criticality for rule to check the fastener length is same as thickness of stack parts
    • Type: string
    • Default value: High
    • Values:
      High Higher criticality
      Medium Moderate criticality
      Low Lower criticality
  • FstFPCNCompass
    Role: :Enable rule to check the fastener positioned at common parent node of stacking
    • Type: boolean
    • Default value: false
  • FstFPCNCompassCriticality
    Role: Criticality for rule to check the fastener positioned at common parent node of stacking
    • Type: string
    • Default value: Medium
    • Values:
      High Higher criticality
      Medium Moderate criticality
      Low Lower criticality
  • FstMSPCompass
    Role: :Enable rule to check the distance between the current fastener and the other ones is greater than a specified distance
    • Type: boolean
    • Default value: false
  • FstMSPCompassCriticality
    Role: Criticality for rule to check the distance between the current fastener and the other ones is greater than a specified distance
    • Type: string
    • Default value: Low
    • Values:
      High Higher criticality
      Medium Moderate criticality
      Low Lower criticality
  • FstNCPPCompass
    Role: :Enable rule to check the fastener normal consistency with the piloting part stacking
    • Type: boolean
    • Default value: false
  • FstNCPPCompassCriticality
    Role: Criticality for rule to check the fastener normal consistency with the piloting part stacking
    • Type: string
    • Default value: High
    • Values:
      High Higher criticality
      Medium Moderate criticality
      Low Lower criticality
  • FstSTKCompass
    Role: :Enable rule to check the consistency between the position of the fastener and its associated fastened parts
    • Type: boolean
    • Default value: false
  • FstSTKCompassCriticality
    Role: Criticality for rule to check the consistency between the position of the fastener and its associated fastened parts
    • Type: string
    • Default value: High
    • Values:
      High Higher criticality
      Medium Moderate criticality
      Low Lower criticality
  • FstSTKSuppositionMode
    Role: Indicates which mode of supposition to be considerd for STK rule computation
    • Type: integer
    • Default value: 1
    • Values:
      1 Stacking Mode: Fastener Stacking is incorrect
      2 Location Mode: Fastener location is incorrect
      3 Stacking Or Location Mode: Either of Stacking or Location is incorrect
