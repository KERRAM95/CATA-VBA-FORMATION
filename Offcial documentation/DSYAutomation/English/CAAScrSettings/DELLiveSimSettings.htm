DELLiveSimSettings Setting Repository
Quick Reference

DELLiveSimSettings

Role: Settings to define the appearance of Live Simulation environment

Detailed role: These settings allow users to customize the appearance of Live Simulation environment

The DELLiveSimSettings repository has the following characteristics:

  • Primary Tab Page: /Infrastructure/DELMIA Infrastructure/Live Simulation
  • Repository File: DELLiveSimSettings
  • Version: 14
  • Full Compatibility: 1
  • Upward Compatibility: 1

DELLiveSimSettings setting repository attributes:

  • ASOLowLight
    Role: Low-light objects not related to ASO
    Detailed role: Indicates if objects not related to ASO should be low-lighted or not
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Disable ASO low-light
      1 Enable ASO low-light
  • ActivateAfterCE
    Role: Activate State after creation/modification
    Detailed role: Indicates if state is to be set active after creation/modification
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Do not set state active after creation/modification
      1 Set state active after creation/modification
  • AdvExportCGR
    Role: Display option panel for Export as CGR command
    Detailed role: Indicates if the options panel is displayed or not for Export as CGR command
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Do not display option panel for Export as CGR command
      1 Display option panel for Export as CGR command
  • AppEscApplySimState
    Role: Execute an application escape on applying a simulation state
    Detailed role: Indicates if an application escape is executed when applying a simulation state
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Do not execute an application escape on applying a simulation state
      1 Execute an application escape on applying a simulation state
  • AppEscInsertNewData
    Role: Execute an application escape on inserting new data
    Detailed role: Indicates if an application escape is executed when inserting new data
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Do not execute an application escape on inserting new data
      1 Execute an application escape on inserting new data
  • AppEscSimEvent
    Role: Execute an application escape on certain simulation event
    Detailed role: Indicates if an application escape is executed when certain even happens during simulation
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Do not execute an application escape on certain simulation event
      1 Execute an application escape on certain simulation event
  • ApplyStateAfterCE
    Role: Apply State after creation/edition
    Detailed role: Indicates if state is applied after creation/edition
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Do not apply state after creation/edition
      1 Apply state after creation/edition
  • ApplyStateCol
    Role: Apply Stored Color/Opacity
    Detailed role: Indicates if color/opacity values stored in states are applied
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Disable color/opacity application
      1 Enable color/opacity application
  • ApplyStateCtx
    Role: Manages the Apply State Context
    Detailed role: Indicates the context to be used while applying a simulation state
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Global (all)
      1 ASO
  • ApplyStateDOF
    Role: Apply Stored DOF
    Detailed role: Indicates if DOF values stored in states are applied
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Disable DOF application
      1 Enable DOF application
  • ApplyStateInSim
    Role: Apply Active State during Simulation
    Detailed role: Indicates if active state is applied during simulation
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Disable active state application during simulation
      1 Enable active state application during simulation
  • ApplyStatePos
    Role: Apply Stored Position
    Detailed role: Indicates if position values stored in states are applied
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Disable position application
      1 Enable position application
  • ApplyStateScope
    Role: Manages the Apply State Scope
    Detailed role: Indicates the scope to be used while applying a simulation state
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Nominal + Overloads
      1 Overloads only
  • ApplyStateViewpoint
    Role: Apply Stored Viewpoint
    Detailed role: Indicates if viewpoint parameters stored in states are applied
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Disable viewpoint application
      1 Enable viewpoint application
  • ApplyStateVis
    Role: Apply Stored Visibility
    Detailed role: Indicates if visibility values stored in states are applied
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Disable visibility application
      1 Enable visibility application
  • BackgrColor
    Role: Color used for 3D Simulation background
    • Type: unsigned integer
    • Default value: 64
    • Min value: 0
    • Max value: 255
  • DROTabDisplay
    Role: Data Readout Tabular Display
    Detailed role: Indicates if tabular display is used for Data Readout
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 No tabular display
      1 Tabular display
  • DesignInContext
    Role: Manages Design in Context option
    Detailed role: Indicates if Design in Context option is enabled or not
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Disable Design in Context
      1 Enable Design in Context
  • FixedInstPosSupport
    Role: Fixed instance position support in Live Simulation.
    • Type: unsigned integer
    • Default value: 0
    • Default Value Role: Disabled
    • Values:
      0 Disabled
      1 Verbose
      2 Enabled
  • FloorManagement
    Role: [Deprecated R2016x]Manages position of the floor
    Detailed role: Indicates where is the floor positioned
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Fixed
      Detailed role: Floor always fixed at 0
      1 Variable
      Detailed role: Floor always placed underneath the existing geometry
  • KeepTreeVis
    Role: Keep tree visibility while entering/leaving 3D Simulation
    Detailed role: Indicates if visibility of the tree should be kept while entering/leaving 3D Simulation
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Do not keep tree visibility
      1 Keep tree visibility
  • ModifyAtomicStateName
    Role: [Deprecated R2018x]Modify Atomic State Name At Creation
    Detailed role: Indicates if users can modify atomic state names at creation
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Disable modification of atomic state name
      1 Enable modification of atomic state name
  • PersistRscGP
    Role: Manages Persist Resource Graphical Properties option
    Detailed role: Indicates if Persist Resource Graphical Properties option is enabled or not
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Do not persist resource graphical properties
      1 Persist resource graphical properties
  • PersistRscPosition
    Role: Manages Persist Resource Position option
    Detailed role: Indicates if Persist Resource Position option is enabled or not
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Do not persist resource position
      1 Persist resource position
  • RecordStateCol
    Role: Record Color/Opacity during state creation
    Detailed role: Indicates if color/opacity information is recorded during simulation state creation
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Color/opacity not recorded
      1 Color/opacity is recorded
  • RecordStateDOF
    Role: Record DOF during state creation
    Detailed role: Indicates if DOF information is recorded during simulation state creation
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 DOF not recorded
      1 DOF is recorded
  • RecordStatePos
    Role: Record Position during state creation
    Detailed role: Indicates if position information is recorded during simulation state creation
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Position not recorded
      1 Position is recorded
  • RecordStateViewpoint
    Role: Record Viewpoint during state creation
    Detailed role: Indicates if viewpoint information is recorded during simulation state creation
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Viewpoint not recorded
      1 Viewpoint is recorded
  • RecordStateVis
    Role: Record Visibility during state creation
    Detailed role: Indicates if visibility information is recorded during simulation state creation
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Visibility not recorded
      1 Visibility is recorded
  • SessionRestoreEnter
    Role: Manage Session Restore information while entering Live Simulation.
    • Type: unsigned integer
    • Default value: 2
    • Default Value Role: Never restore current state
    • Values:
      0 Always restore current state (if available)
      1 Ask before restoring current state
      2 Never restore current state
  • SessionRestoreExit
    Role: Manage Session Restore information while exiting Live Simulation.
    • Type: unsigned integer
    • Default value: 2
    • Default Value Role: Never save current state
    • Values:
      0 Always save current state
      1 Ask before saving current state
      2 Never save current state
  • SimStateInPrd
    Role: Indicates if simulation states are allowed under product
    Detailed role: Indicates if simulation states can be created under product
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Do not create simulation states under product node
      1 Create simulation states under product node
  • ThumbnailGrouping
    Role: Group thumbnails in immersive browser tabs
    Detailed role: Indicates if thumbnails displayed in immersive browser tabs should be grouped or not
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Disable thumbnail grouping
      1 Enable thumbnail grouping
  • ThumbnailSize
    Role: Manages the size of thumbnails
    Detailed role: Indicates the size of thumbnails displayed in immersive browser tabs
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Small
      1 Medium
      2 Large
