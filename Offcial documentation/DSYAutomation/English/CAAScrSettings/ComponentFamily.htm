ComponentFamily Setting Repository
Quick Reference

ComponentFamily

Role: Settings to manage component family specific connection used in items resolution

The ComponentFamily repository has the following characteristics:

  • Primary Tab Page: /MechanicalDesign/ComponentFamily
  • Repository File: ComponentFamily
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

ComponentFamily setting repository attributes:

  • CfyInitializeWithMinimumDuplicateConfig
    Role: Defines if the specific option 'Define Components Not To Duplicate' to be activated
    • Type: boolean
    • Default value: false
  • CfyLockDefineComponentsNotToDuplicate
    Role: Locks the default settings provided by 'Define Components Not To Duplicate'
    • Type: boolean
    • Default value: false
  • CfyUseItemsOnlyWithTargetMaturity
    Role: Defines if the specific option 'Override item with Target Maturity' to be activated
    • Type: boolean
    • Default value: false
  • ExternalAuthentificationMode
    Role: Defines if the specific connection described in this settings should be used
    • Type: boolean
    • Default value: false
  • servername
    Role: Defines the server to be used to resolve family items
    • Type: URN
    • Default value:
  • serverport
    Role: Defines the port of the server to be used to resolve family items
    • Type: integer
    • Default value: 0
  • serverprotocol
    Role: Defines the protocol of the server to be used to resolve family items
    • Type: integer
    • Default value: 0
  • serveruri
    Role: Defines the URI of the server to be used to resolve family items
    • Type: URN
    • Default value:
  • username
    Role: Defines the user to be used to resolve family items
    • Type: URN
    • Default value:
  • userpassword
    Role: Defines the user password to be used to resolve family items
    • Type: URN
    • Default value:
  • usersecuritycontext
    Role: Defines the security context of the user to be used to resolve family items
    • Type: URN
    • Default value:
